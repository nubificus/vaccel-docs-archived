    vAccel on AWS Firecracker micro-VMs

    AWS Firecracker is a Virtual Machine Monitor (VMM) that allows creating lightweight Virtual Machines with extermely small boot times. Firecracker powers the AWS Lambda serverless infrastructure and it's used in various use cases where the isolation of Virtual Machines along with efficiency of containers is needed.

    For example, Kata containers incorporate Firecracker to allow booting isolated containers with minimum overhead.

    We have extended to add a new virtio device implementation for our virtio-accel module. Our implementation is on top of v0.23.1 release of upstream Firecracker (branch vaccel-v0.23.1 on our forked repo).

    Running vAccel enabled Firecracker

    Building our forked Firecracker is equivalent to building upstream Firecracker, with the only exception that we have to build Firecracker linking against standard GNU libc, instead of musl.

    git clone https://github.com/cloudkernels/firecracker vaccel-v0.23.1
./tools/devtool build -l gnu

    The Firecracker binary is located under ./firecracker/build/cargo_target/x86_64-unknown-linux-gnu/debug/firecracker.

    Now, Firecracker is launched like any other vAccel application. Additionally, we need to provide a configuration file for our Firecracker VM. We have extended the spec to include a vaccel section. Here is an example:

    {
    "boot-source": {
        "kernel_image_path": "share/vmlinux",
        "boot_args": "console=ttyS0 reboot=k panic=1 pci=off loglevel=8 root=/dev/vda ip=172.42.0.2::172.42.0.1:255.255.255.0::eth0:off random.trust_cpu=on"
    },
    "drives": [
        {
            "drive_id": "rootfs",
            "path_on_host": "share/rootfs.img",
            "is_root_device": true,
            "is_read_only": false
        }
    ],
    "network-interfaces": [
    {
        "iface_id": "eth0",
        "guest_mac": "AA:FC:00:00:00:01",
        "host_dev_name": "tapTestFc"
    }
    ],
    "crypto": {
        "crypto_dev_id": "vaccel0",
        "host_crypto_dev": "/dev/vaccel0"
    },
    "machine-config": {
        "vcpu_count": 2,
        "mem_size_mib": 4096,
        "ht_enabled": false
    }
}

    Notice the crypto section of the json snippet. We still call this crypto because virtio-accel came about as an "extension" of the virtio-crypto device.

    Modify the Firecracker config to match your setup (paths to rootfs and vmlinux, network devices, etc) and boot your Firecracker image like:

    # Set path to libvaccel
export LD_LIBRARY_PATH=/path/to/libvaccel.so

# Determine backend plugin to use
export VACCEL_BACKENDS=/path/to/plugin/libvaccel-jetson.so

# Launch a VM
/path/to/firecracker --api-sock fc.sock --config-file your_fc_config.json --seccomp-level 0

    Note: We need to disable seccomp filters, because plugins such as jetson-inference need to use system calls that normally are not whitelisted by Firecracker.
